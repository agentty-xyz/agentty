# Implementation Plan Skill

Skill instructions for implementation planning tasks.

Keep the `## Priorities` section near the top of each plan, immediately after the title and scope/context line. Do not rename it to `Updated Priorities`.
Priority sections in implementation plans should render `Why now`, `Usable outcome`, `Substeps`, `Tests`, and `Docs` as separate subtopics on their own lines, not as inline bold labels.
Delete fully implemented plan files from `docs/plan/`; if additional work is needed, create a new follow-up plan file instead of extending the completed one.
